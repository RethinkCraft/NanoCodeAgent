# 第 3 章：核心架构设计

## 整体架构

NanoCodeAgent 采用事件驱动的**"感知-推理-行动"**循环（ReAct 模式）：

```
用户输入
   │
   ▼
┌──────────┐
│  CLI     │  解析命令行参数，读取配置
└────┬─────┘
     │
     ▼
┌──────────┐        ┌──────────────┐
│  LLM     │◄──────►│  Tool Router │
│  Client  │        │  (工具路由)  │
└────┬─────┘        └──────┬───────┘
     │                     │
     │              ┌──────┴──────────────┐
     │              │                     │
     ▼              ▼                     ▼
  模型推理      Bash Tool          File Tools
 (流式 SSE)   (Shell 执行)     (读文件/写文件)
```

## 主要模块

### `main.cpp` — 程序入口

负责参数解析、配置加载和主循环启动。

### `llm.cpp` — LLM 客户端

封装对 LLM API 的 HTTP 请求，支持：
- 流式输出（SSE）
- 工具调用结果回传
- 多轮对话历史管理

### `tool_call_assembler.cpp` — 工具调用组装器

将 LLM 流式输出中的工具调用片段组装成完整的调用请求。

### `workspace.cpp` — 工作区管理

维护代理的工作目录，确保文件操作限制在沙盒范围内。

## 关键数据流

1. 用户发送消息 → CLI 接收
2. LLM Client 将消息连同工具描述发送给模型
3. 模型以 SSE 流返回推理结果
4. SSE Parser 解析流，Tool Call Assembler 收集工具调用
5. Tool Router 分发到对应工具执行
6. 工具结果作为新消息追加到上下文
7. 重复 2-6 直到模型返回最终回复
