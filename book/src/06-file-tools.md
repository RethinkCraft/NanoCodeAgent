# 第 6 章：文件系统工具

## 读文件工具（`read_file.cpp`）

### 功能

读取工作区内指定路径的文件内容，返回文本字符串供 LLM 分析。

### 路径安全校验

为防止路径穿越攻击（Path Traversal），读文件工具：
1. 拒绝以 `/` 开头的绝对路径
2. 拒绝包含 `..` 的路径片段
3. 将路径解析到工作区目录下，并验证最终路径仍在工作区内

### 使用示例（LLM 视角）

```json
{
  "name": "read_file",
  "arguments": "{\"path\": \"src/main.cpp\"}"
}
```

返回值为文件内容字符串，若文件不存在则返回错误信息。

## 写文件工具（`write_file.cpp`）

### 功能

在工作区内创建或覆盖文件，支持创建中间目录。

### 路径安全校验

与读文件工具相同的校验逻辑，拒绝危险路径。

### 使用示例（LLM 视角）

```json
{
  "name": "write_file",
  "arguments": "{\"path\": \"output/result.txt\", \"content\": \"Hello, World!\"}"
}
```

### 注意事项

- 写入操作**不可撤销**，LLM 在执行前应谨慎
- 建议在修改已有文件前先用 `read_file` 读取原内容
- 默认单次写入的文件大小上限为 4MB（由 `max_write_bytes` 参数强制限制），在此范围内仍建议控制在 1MB 以内以提高性能和稳定性
